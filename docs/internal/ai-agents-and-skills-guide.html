A Simple Guide

AI Agents & Skills

How they work, how to create them, and why they matter.

Concept

What Is an AI Agent?

An AI agent is a digital teammate that lives inside your development environment. It doesn't just answer questions — it takes action: reads files, runs commands, searches code, and makes changes.

These agents are built into the tools you already use. The concepts of rules and skills work across multiple AI coding tools — the patterns are the same, even if the setup differs slightly.

⚡

Warp (Oz)

AI agent built into the Warp terminal. Works alongside you in the command line — runs commands, edits files, and navigates your codebase.

💬

Claude (claude.ai)

Anthropic's AI assistant. Use Projects to set persistent instructions and upload knowledge files — Claude's version of rules.

🤖

Claude Code

Agentic coding tool that runs in your terminal. Reads your codebase, makes edits, runs tests, and handles git workflows.

Analogy

Without an agent, AI is a consultant who gives advice but never touches the keyboard. With an agent, AI becomes a hands-on partner that does the work with you.

Getting Started

Setting Up Your Agent

Each tool has its own setup, but they all follow the same idea: install the tool, then start talking to it.

⚡

Warp (Oz)

1. Download warp.dev and open it.
2. Navigate to any Git repo — Warp auto-indexes your codebase.
3. Start typing in the prompt bar. Oz is already active.
4. Use /init to generate an AGENTS.md rules file for your project.

💬

Claude (claude.ai)

1. Go to claude.ai and sign in.
2. Create a Project — this is your workspace.
3. Add Project Instructions (your rules) and upload Knowledge files.
4. Start a conversation inside that Project — Claude follows your instructions automatically.

🤖

Claude Code

1. Install: npm install -g @anthropic-ai/claude-code
2. Run claude in any Git repo.
3. On first run, it scans your codebase automatically.
4. Ask it: "Create a CLAUDE.md rules file for this project."

Key Point

You don't need to configure anything complex. Install the tool, open a project, and start talking. The agent is ready from the first prompt.

Foundation

What Are Rules?

Rules are persistent instructions that tell the agent how to behave — across every conversation, every task. Set them once; they apply always.

🌐

Global Rules

Apply everywhere, in every project. Example: "Always use Python 3.11" or "Never push without permission."

📁

Project Rules

Live inside a specific repo. Example: "This project uses Django" or "Run pytest before committing."

Warp (Oz)

Global: Warp Drive → Rules

Project: AGENTS.md in repo root

Claude (claude.ai)

Project Instructions (set per Project)

Knowledge files uploaded to a Project

Claude Code

Global: ~/.claude/CLAUDE.md

Project: CLAUDE.md in repo root

Analogy

Rules are the house rules you give a new roommate. They don't need to ask every time — they just know.

Core Concept

What Are Skills?

Skills are reusable playbooks that teach the agent how to do a specific task. While rules say "always behave this way," skills say "when asked to do X, follow these exact steps."

Each skill is a simple markdown file (SKILL.md) containing a name, description, and step-by-step instructions.

✉️

Customer Email

Writes emails in a professional, jargon-free tone for non-technical customers.

🔍

Tenant Lookup

Walks through looking up tenant info from Django admin step by step.

🚀

Create PR

Builds a structured pull request description from branch context and Jira tickets.

Analogy

If rules are house rules, skills are recipes. Write the recipe once, and the agent can cook that dish any time you ask.

Comparison

Rules vs Skills — When to Use Which?

They complement each other. Rules define the boundaries; skills define the workflows.

📐 Rules

  • Coding standards & preferences
  • Behavior constraints ("never do X")
  • Project conventions
  • Communication style
  • Always active, no invocation needed

⚡ Skills

  • Multi-step workflows
  • Repeatable procedures
  • Domain-specific playbooks
  • Can include scripts & templates
  • Invoked on demand or auto-detected
How-To

How to Create Rules

You don't need to write config files by hand. Just ask the agent to do it for you.

⚡

In Warp

Type /init to auto-generate an AGENTS.md for your project. Or ask:

"Add a rule that says: always use Python 3.11 and never push to remote without asking."

💬

In Claude (claude.ai)

Open a Project, click Project Instructions, and type your guidelines in plain English. No special format needed.

"Always respond in a professional but friendly tone. Avoid technical jargon."

🤖

In Claude Code

Just ask it directly in your terminal:

"Create a CLAUDE.md file for this project with rules for our Django codebase."

Tip

Write rules like you're onboarding a new teammate. Plain language works best — no special syntax required.

How-To

How to Create a Skill

The easiest way? Ask the agent to create the skill for you. Describe the workflow you want automated, and the agent writes the skill file.

⚡

Prompt Warp

"Create a new skill called 'customer-email' that writes professional, jargon-free emails to customers. Save it in .agents/skills/."

Warp creates the folder, writes the SKILL.md, and it's ready to use immediately.

💬

Prompt Claude (claude.ai)

"Write me a reusable prompt template for writing customer emails in a friendly, non-technical tone."

Save Claude's output as a Knowledge file in your Project. It works like a skill — reference it by name in future chats.

🤖

Prompt Claude Code

"Create a skill called 'create-pr' that builds structured PR descriptions from the current branch and Jira ticket. Put it in .claude/skills/."

Claude Code scaffolds the folder and SKILL.md in your repo.

Key Point

You describe what the skill should do. The agent handles the how — file structure, formatting, and placement.

Example

What a Skill File Looks Like

This is all it takes — a single markdown file with frontmatter and clear instructions.

SKILL.MD --- name: add-feature-flag description: Add a new feature flag with proper config and docs --- # Add Feature Flag ## Instructions 1. Ask the user for the flag name and default value 2. Add the definition to `config/feature_flags.yaml` 3. Create a helper function in `src/utils/flags.ts` 4. Update documentation in `docs/FEATURE_FLAGS.md` ## Examples - "Add a feature flag for the new checkout flow" - "Create a flag to enable dark mode for beta users"
Why It Matters

The Difference Agents & Skills Make

Without them, every AI interaction starts from zero. With them, your AI remembers, knows, and acts.

❌ Without Agents + Skills

  • Re-explain context every conversation
  • Repeat preferences and constraints
  • Copy-paste outputs manually
  • Walk through steps one by one
  • AI advises but never executes

✅ With Agents + Skills

  • Rules carry context automatically
  • Skills encode entire workflows
  • Agent reads, writes, and runs code
  • Complex tasks become one command
  • AI is a hands-on teammate
